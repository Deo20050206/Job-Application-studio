# Docs

This folder contains the public-facing explanation layer for the repository.

It is organized to show three things clearly:

- the framework behind how I use AI
- the workflows I use in practice
- a concrete case study that shows those ideas in action

## Structure

- `frameworks/`
  AI principles, operating model, and system design thinking.
- `workflows/`
  How AI is used in day-to-day execution, analysis, and project building.
- `case-studies/`
  A real example of the methodology applied inside this workspace.

## Reading Order

1. `frameworks/ai-operating-model.md`
2. `frameworks/ai-system-design-principles.md`
3. `workflows/daily-ai-operations.md`
4. `workflows/ai-project-building.md`
5. `workflows/tooling-and-systems.md`
6. `case-studies/job-application-studio.md`
