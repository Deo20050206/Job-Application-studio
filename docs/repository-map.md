# Repository Map

## Purpose

This file gives a quick explanation of what each major part of the repository is responsible for.

## Top-Level Structure

- `docs/`
  Public-facing explanation of methodology, workflows, and case studies.
- `inputs/`
  Private source materials and truth boundaries.
- `jobs/`
  Structured intake files for incoming work items.
- `outputs/`
  Final deliverables and public placeholders.
- `skills/`
  Custom AI behavior definitions.
- `templates/`
  Base structures used to keep outputs consistent.
- `tmp/`
  Working helpers, draft scripts, and iterative tooling.
- `tracker/`
  Status tracking and visibility over ongoing work.

## Current Showcase Logic

The repository is intentionally shaped to let a GitHub visitor understand:

- what the workflow is
- how AI fits into it
- where the safeguards live
- which parts are public
- which parts are intentionally excluded
