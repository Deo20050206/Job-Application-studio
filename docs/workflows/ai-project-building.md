# AI Project Building

## How AI Helps Me Build Projects

I use AI to accelerate project construction at the system level, not only at the content level.

That usually means using AI to help with:

- repo structure
- workflow design
- prompt and skill design
- internal documentation
- boilerplate generation
- iteration on process quality

## Build Pattern

The pattern I follow is usually:

1. define the outcome
2. map the inputs
3. define rules and constraints
4. create reusable templates
5. add a quality-control layer
6. keep private data separate from the reusable system

## What This Repository Shows

This repository demonstrates that pattern through a real workflow:

- `WORKFLOW_RULES.md` defines system constraints
- `generation_checklist.md` defines QA gates
- `skills/` shows custom AI behavior design
- `templates/` shows reusable project scaffolding
- `jobs/`, `outputs/`, and `tracker/` show how work moves through the system

The value is in making the process legible, reusable, and disciplined.
